# **DC (Distributed Computing) Project Summary**

## **🎯 Mission & Objectives**

The **DC** project aims to build a high-performance, **Peer-to-Peer (P2P) Distributed Computing framework**

Our core objectives are:

1. **Decentralized Computation**: Minimize reliance on a central server by making task distribution, reservation, and execution happen directly between peers.
2. **Network Awareness**: Optimize task placement by measuring and prioritizing peer-to-peer latencies, ensuring work goes to the most responsive nodes.
3. **Resilient Fault Tolerance**: Implement a robust Primary-Backup replication system to ensure computation integrity even when nodes fail mid-task.
4. **Heterogeneous support**

## **🏗️ Architecture Overview**

The system follows a hybrid P2P architecture:

- **Coordinator**: A lightweight service (registry) that handles node discovery and maintains fault-tolerance metadata (task replicas). It does *not* participate in computation.
- **Worker Nodes**: The core of the swarm. Each node can act as:
    - **Client**: Partitioning large tasks and dispatching them to available peers.
    - **Worker**: Performing actual computations
- **Network-Aware Routing**: Nodes measure peer latencies and prioritize "good" connections.
- **Fault Tolerance**: Implements Primary-Backup replication. If a primary node fails, the coordinator can promote the backup
